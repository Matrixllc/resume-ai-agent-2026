# 通用简历标准

适用关键词：通用、默认、综合、简历标准、简历优先级、岗位未命中

关注领域：不限领域，按简历事实完整度和岗位相关证据排序。

核心技能：
- 技能标签清晰，能覆盖目标岗位所需的主要技术或业务能力。
- 项目、工作经历、教育背景之间逻辑一致。
- 有可复核的项目职责、技术栈、业务场景、结果指标或交付成果。

项目证据信号：
- 项目描述包含明确问题、本人职责、使用方法、产出结果。
- 能体现复杂度，如跨团队协作、系统设计、数据分析、流程优化、业务落地。
- 有量化结果、上线经历、稳定运行、效率提升、成本降低等事实。

风险信号：
- 缺少项目级证据。
- 技能标签很多但项目无法支撑。
- 岗位相关经历不足或描述过于笼统。
- 关键信息缺失，无法确认真实贡献。

# 金融岗位标准

适用关键词：金融、金融分析、风控、交易、投研、支付、量化、股票、基金、信贷

关注领域：Finance、金融业务、风险控制、交易系统、投研分析、支付清算。

核心技能：
- Python、SQL、数据分析、统计分析、报表分析。
- 金融业务理解，包括风控、交易、投研、支付、量化、信贷。
- 能将业务指标、风险指标或交易指标落到数据处理和分析结论。

项目证据信号：
- 金融项目、交易系统、风控模型、投研分析、量化策略、支付或信贷项目。
- 使用 SQL/Python/pandas/numpy 等工具处理金融数据。
- 有风险识别、策略评估、收益回测、指标监控或业务复盘证据。

风险信号：
- 只有金融关键词，没有金融项目或数据证据。
- 只有工具技能，缺少金融业务理解。
- 没有说明分析结论、指标或业务影响。

# 运营岗位标准

适用关键词：运营、用户运营、活动运营、增长、商家运营、会员运营、内容运营、策略运营

关注领域：Operations、用户增长、活动运营、商家运营、会员运营、数据复盘。

核心技能：
- 用户分层、活动策划、增长策略、数据分析、复盘迭代。
- 跨团队协作、资源协调、执行推进、结果跟踪。
- 能用指标解释运营动作的效果。

项目证据信号：
- 用户增长、活动转化、会员体系、商家运营、内容运营、策略运营项目。
- 有拉新、促活、留存、转化、GMV、复购、活跃度等指标。
- 有策略设计、实验对比、数据复盘、流程优化或运营 SOP。

风险信号：
- 只有运营岗位名称，缺少指标和项目过程。
- 只描述执行动作，没有结果复盘。
- 缺少数据分析或协作证据。

# 能源岗位标准

适用关键词：能源、新能源、电力、储能、风电、光伏、碳资产、碳交易、电力交易、能源项目

关注领域：Energy、新能源、电力交易、储能、风电、光伏、碳资产、能源项目管理。

核心技能：
- 能源行业知识、电力市场、项目管理、政策理解、数据分析。
- 了解新能源、电力交易、储能、风电、光伏、碳资产相关业务流程。
- 能处理项目推进、方案评估、市场分析或运营管理问题。

项目证据信号：
- 电力交易、储能产品、风电运维、碳资产、能源项目管理经历。
- 有政策解读、市场分析、收益测算、项目排期、风险控制或交付结果。
- 能说明能源业务场景和本人负责环节。

风险信号：
- 只有行业标签，缺少能源项目细节。
- 缺少政策、市场、项目或技术场景证据。
- 无法确认本人贡献和业务结果。

# 后端开发岗位标准

适用关键词：后端、后端开发、服务端、Backend、Java 后端、Golang 后端、Node.js 后端、Python 后端

关注领域：后端开发、服务端工程、系统设计、数据库、中间件、接口与稳定性。

核心技能：
- Java、Golang、Python、Node.js、Spring Boot、MySQL、Redis、MongoDB、Kafka、SQL。
- 接口设计、数据库设计、缓存、消息队列、并发处理、服务稳定性、性能优化。
- 工程化能力，包括代码可维护性、部署、监控、问题排查、文档沉淀。

项目证据信号：
- 后端服务、生产系统、搜索/推荐工程、数据服务、接口系统、消息队列、线程池、缓存、数据库项目。
- 有系统设计、表结构设计、接口设计、异步化、解耦、性能优化、稳定性治理证据。
- 有上线、迁移、调优、故障排查或效率提升结果。

风险信号：
- 只有编程语言标签，缺少后端项目证据。
- 项目只停留在脚本或单机应用，缺少服务端复杂度。
- 没有稳定性、性能、接口、数据库或中间件相关事实。

# AI/AI Search 岗位标准

适用关键词：AI、AI Search、搜索、推荐、召回、排序、RAG、LLM、向量检索、Agent、机器学习

关注领域：AI Search、搜索推荐、RAG、LLM 应用、向量检索、模型或算法工程落地。

核心技能：
- Python、RAG、LLM、向量检索、Embedding、搜索系统、推荐系统、召回、排序。
- 能理解模型、检索、排序、评估指标和工程落地之间的关系。
- 能与算法、工程、产品或业务团队协作推进 AI 项目。

项目证据信号：
- 搜索、推荐、召回、排序、RAG、LLM、向量检索、机器学习项目。
- 有模型特征、检索策略、排序优化、评估指标、效果提升或工程部署证据。
- 能说明 AI 能力如何落到真实业务流程。

风险信号：
- 只有 AI 关键词，缺少项目落地证据。
- 只描述工具或框架，缺少业务效果和评估方式。
- 缺少工程化、数据、模型或检索链路证据。
